About Me

I am Sebastian Alvis, a 24-year-old coding enthusiast, cat lover, and dungeon master. I earned a Bachelor's in Engineering Physics at UIUC and a Master's Degree in Physics at UW, then went on to complete a 6-month, 500-hour data science bootcamp from SpringBoard. Now, I am looking to start my data science career. I like knowing how things work together, and being able to own the data science process end-to-end.

Code Used in the Data Science Bootcamp

The bootcamp refined my Python and SQL skills, and taught me various Python packages for use in data science and machine learning, such as Pandas, scikit-learn, keras, and more. Most of the work I submitted for grading was through GitHub, and the two capstone projects I did were in Jupyter Notebooks.

The two capstone projects I did are available on GitHub here.

The first project is an analysis of game data from League of Legends. I act as if I am an analyst for Riot Games, the studio that makes this game, trying to help keep the game balanced. At the end of the project, I make 8 strong balance recommendations that I found from my statistical and machine learning analyses. I gave this final presentation about this work at the PuPPy Scientific Computing meeting on April 25th, 2019. More information is available in this repository.

The second project is an analysis of Twitter data about the game studio, Bungie, and their hit game, Destiny 2. In this, I act as an analyst trying to glean any information I can on the public's opinion of the studio and the game. I analyze the habits of those on Twitter and find the factors that make tweets popular, as well as discovering that positive sentiment tweets get more engagement than negative ones. This shows that the general audience of Destiny 2 thinks well of the studio and game. More information about this project can be found here.

Code Used in Research

The majority of my year and a half in the Majorana group has been spent learning ROOT. This toolkit can be mounted onto Python and C++, both of which I used to run ROOT scripts (with the help of some CSH) on the PDSF supercomputer at UC Berkeley.

Within Python, I used the numpy and matplotlib modules, and also created my own modules for reusability and the ease of merging existing code into a single script.

The only non-ROOT libraries in C++ that I used was the STL, for the map and vector data structures.

Research Experience

Experimental physics research is heavily multidisciplinary. My year and a half with the Majorana experiment involved engineering, programming, and data science.

Engineering

There is a germanium detector at UW of the same style that the Majorana experiment uses. In working to bring this detector to working capacity, I used Autodesk Inventor to design, install, and test three-dimensional parts. I have maintained this detector’s assembly, repaired the turbopump it uses, and began redesigning it to incorporate suggested changes from a report on a variable-position alpha-source scanner.

Programming

The primary software framework used in experimental physics is ROOT. ROOT features unique data structures, improves parallelization on processing, scanning, and moving large amounts of data, and has tools to visualize and analyze data. I learned ROOT mostly by myself and with some help from those in my research group, and used it to learn Majorana’s data and the analysis tasks I performed. I mounted ROOT onto Python and C++ and built modules and scripts to parallelize my data processing and to perfect my data-quality-checking workflow.

I gave a talk about this work at the PuPPy Scientific Computing meeting on Aug 16th, 2018.

Data Science

My initial analysis consisted of tracking a parameter across detectors and time, finding areas where it was unstable, and fixing them using existing scripts, which required occasional debugging and modification. Later analyses performed an instability quantification, calculating the parameter shift that encapsulated a few percent of the data. This was used as a benchmark to validate instability levels.
